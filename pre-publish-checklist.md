# 视频自动化发布前检查清单

> 全自动发布前做一次快速检查，避免「翻车式发布」。

## 为什么需要这个清单

AI 全自动流水线的最大风险不是质量低，而是 **翻车不被察觉**：
- 语音输出变成了火星文
- 画面和文案完全不搭
- 视频出现了脏话/敏感词（AI 幻觉）
- 发布了但平台限流都没人知道

这个清单不是每个视频都跑一遍，而是 **发布前脚本自动执行一次质量门禁**。

## 一、文案质量门禁（自动检查）

由脚本自动扫描，不通过则标记人工审核或直接拦截。

### 1.1 敏感词检测

```
检测项：政治敏感词 / 违禁词 / 竞品贬低 / 不实宣传
处理规则：
- 命中红色级别 → 拦截，不发布
- 命中黄色级别 → 标记，人工确认
- 无命中 → 通过
```

常见翻车点：
- 「最」「第一」「绝对」等极限词（广告法）
- 未经验证的医疗/金融建议
- 贬低其他平台或竞品
- 抄袭/洗稿风险（可接入查重 API）

### 1.2 文案结构检测

```
检测项：
- 是否有钩子在前 20% 篇幅内？        [是/否]
- 是否有明确的结尾/CTA？            [是/否]
- 是否有 3 个以上独立信息点？        [是/否]
- 全文是否 > 30 字？                [是/否]
- 全文是否 < 平台最大时长对应字数？   [是/否]
```

规则建议：5 项中至少 3 项通过，否则标记人工审核。

### 1.3 口语化检测

```
检测项：
- 是否有书面语（「然而」「因此」「该产品」等）？
- 平均句长是否 > 20 字？
- 是否有 AI 常见套话（「值得注意的是」「在当今时代」）？
```

任一命中 → 标记建议修改，不强制拦截（不同平台接受度不同）。

## 二、音频质量门禁（自动检查）

### 2.1 TTS 输出检查

```
检测项：
- 音频时长是否在预期 ±20% 内？     [检查]
- 音频文件是否可解码？              [检查]
- 是否有长时间静音（> 2s）？         [检查]
- 采样率和格式是否与视频引擎兼容？   [检查]
```

静音检测特别重要——AI TTS 有时在生僻词前后输出静音片段。

### 2.2 语速检查

```
目标语速：平台适配（抖音 280-320字/分，B站 220-260，视频号 200-240）
检查：
- 实际语速是否在目标范围 ±15% 内？
- 超速（> 350 字/分）→ 建议降速或删内容
- 过慢（< 150 字/分）→ 建议补充内容
```

## 三、视频质量门禁（自动检查）

### 3.1 画面检测

```
检测项：
- 视频是否有黑帧？                  [检查前 3s + 最后 3s]
- 视频分辨率是否达标？              [目标 ≥ 1080p]
- 画面比例是否正确？                [抖音 9:16 / B站 16:9 / 视频号 9:16]
- 是否有画面卡顿或跳帧？            [检查帧率一致性]
```

### 3.2 画音同步检测

```
检测项：
- 音频时长和视频时长是否匹配？（±1s） [检查]
- 字幕和音频是否同步？             [检查时间戳对齐]
```

画音不同步是自动化流水线最常见的翻车问题——字幕引擎和视频引擎时间轴不一致导致。

### 3.3 字幕/水印检查

```
检测项：
- 字幕是否越界（超出画面安全区）？     [检查]
- 字幕是否有乱码/特殊字符？           [检查]
- 水印位置是否在安全区内？            [检查]
- 是否有竞品水印？（如另一平台的 Logo 误入）
```

## 四、平台适配检查（自动 + 配置驱动）

```
配置示例（每个平台一个 profile）：

抖音：
  - 时长上限：60s（可配置）
  - 比例：9:16
  - 禁用标签：#广告 #推广
  - 标题字数上限：55
  - 需加话题标签：≥3 个

B站：
  - 时长下限：60s
  - 比例：16:9
  - 标题字数上限：30
  - 必须选择分区

视频号：
  - 时长：≤ 15min
  - 比例：9:16
  - 需绑定公众号/企业微信（可选）

YouTube：
  - 时长：≥ 8min（插 mid-roll 广告）
  - 比例：16:9
  - 需填写描述（≥ 200 字 SEO）
  - 需设置缩略图
```

自动检查项：
- [ ] 视频规格符合目标平台 profile
- [ ] 标题长度在平台限制内
- [ ] 标签/话题数符合平台最佳实践
- [ ] 描述/简介按要求填写

## 五、发布前终检（人+机器）

### 自动部分（发布脚本执行）
```
- 文案门禁通过                    [PASS/FAIL]
- 音频门禁通过                    [PASS/FAIL]
- 视频门禁通过                    [PASS/FAIL]
- 平台适配通过                    [PASS/FAIL]
- 总体判定：
  - 全部 PASS → 自动发布
  - ≥1 个 FAIL → 拦截，标记待审核
  - 敏感词命中红色 → 拦截，不发布
```

### 人工部分（每天统一处理拦截项）
```
- 查看拦截的视频列表
- 确认是真的翻车还是误报
- 修复/删除/放行
- 误报的更新规则到门禁配置
```

## 六、发布后监控（可选）

建议对已发布内容做 24h 监控：

| 监控项 | 触发条件 | 动作 |
|--------|---------|------|
| 播放量骤降 | 某视频播放量 < 账号均值 50% | 查看是否限流 |
| 评论区异常 | 出现大量负面/举报 | 自查内容合规性 |
| 平台通知 | 收到违规/限流通知 | 立即下架并分析 |
| 数据异常 | 播放/点赞比 < 2% | 优化钩子/标题 |

## 七、翻车案例库（持续积累）

| 日期 | 翻车类型 | 原因 | 修复方案 |
|------|---------|------|---------|
| — | TTS 吐字错误 | 生僻词/多音字 | 加入生僻词词典，TTS 前替换 |
| — | 画音不同步 | 字幕引擎时间轴偏移 | 使用同一个时间轴源 |
| — | 敏感词拦截 | AI 无意中输出违禁词 | 加强文案门禁的敏感词库 |
| — | 分辨率不足 | 素材源文件太糊 | 设置最低分辨率阈值 |
