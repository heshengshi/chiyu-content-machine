# 内容选题信号源框架

> 系统化发现「值得做」的选题。选题决定上限，执行决定下限。

## 为什么需要这个框架

**问题**：
- 做完一条内容发现没人看——选题本身就没需求
- 看到热点想追，但不知道怎么转化成自己的选题
- 别人能持续产出爆款选题，我却靠灵感碰运气

**目标**：
- 从「灵感驱动」到「信号驱动」
- 建立可持续的选题发现管道
- 每个选题都有「为什么选它」的理由

## 一、选题信号分级

不是所有信号都值得做选题。先分级：

|| 信号级别 | 特征 | 处理方式 |
|---------|------|---------|
| **A级** | 多平台验证 + 数据爆炸 | 立刻做，抢占时间窗口 |
| **B级** | 单平台起量 + 可迁移 | 本周做，适配自己的赛道 |
| **C级** | 小范围讨论 + 有增量 | 做不做看产能，做了有价值 |
| **D级** | 纯情绪/无增量 | 不做 |

**判断标准**：

```
A 级信号 = （播放/互动 ≥ 同期均值 3 倍）+（至少 2 个平台出现）
B 级信号 = （播放/互动 ≥ 同期均值 2 倍）+（1 个平台出现但可迁移）
C 级信号 = （小圈层讨论热）+（有独特信息增量）
D 级信号 = （纯情绪宣泄）或（没有信息增量）
```

## 三、选题转化公式

发现信号后，如何转化为自己的选题？

### 3.1 热点类 → 观点/工具类选题

```
公式：热点事件 + 我的视角/工具 = 我的选题

示例：
- 热点：Claude 3.5 发布
- 我的视角：AI 工具测评者
- 转化：「Claude 3.5 实测：这 3 个功能让 GPT-4 感到压力」
```

### 3.2 需求类 → 教程/工具类选题

```
公式：用户问题 + 解决方案 = 我的选题

示例：
- 需求：群里有人问「怎么做 AI 视频不花时间」
- 解决方案：自动化流水线
- 转化：「全自动 AI 视频流水线搭建教程」
```

### 3.3 信息差类 → 科普/测评类选题

```
公式：圈内已知 + 圈外不知道 = 我的选题

示例：
- 信息差：ClawHub 发现的某工具
- 转化：「国内没人用的这个 AI 工具，我帮你们测了」
```

### 3.4 成功案例类 → 方法论选题

```
公式：别人的成功 + 拆解分析 = 我的选题

示例：
- 案例：某账号 7 天涨粉 10 万
- 转化：「拆解某账号 7 天涨粉 10 万的 5 个关键动作」
```

## 四、选题评估打分卡

每个候选选题用这个打分卡：

| 维度 | 权重 | 1分 | 3分 | 5分 |
|------|------|-----|-----|-----|
| **需求强度** | 30% | 伪需求 | 小众需求 | 刚需/热点 |
| **竞争程度** | 25% | 红海一片 | 有人做但不多 | 蓝海/先发优势 |
| **与我的匹配** | 25% | 完全不相关 | 能硬扯上 | 直接相关 |
| **可执行性** | 20% | 做不了 | 能做但费劲 | 很快能出 |

**判断规则**：
- ≥ 3.5 分 → 必做
- 2.5-3.4 分 → 排进队列
- < 2.5 分 → 放弃或存档观察

## 五、选题生产流水线

```
信号采集 → 信号分级 → 选题转化 → 打分筛选 → 排期生产
   10min      5min       5min       3min      --
```

### 每日操作流程

**早间（9:00-9:30）**：
1. 扫描抖音热榜 + 微博热搜（5 分钟）
2. 扫描 GitHub Trending + ProductHunt（5 分钟）
3. 记录 3-5 个信号到选题池（10 分钟）
4. 选择 1 个立即做或排期

**晚间（21:30-22:00）**：
1. 消化 AI Native 群 + 觅游 + ClawHub（15 分钟）
2. 提取「信号-反应」对，转化为选题（10 分钟）
3. 更新选题池

### 每周操作

1. 复盘本周数据，分析爆款选题共性
2. 清理选题池（过期选题删除，长期选题标记）
3. 规划下周选题排期

## 六、选题池管理

### 选题池格式

```markdown
## 选题池（2026-06-27 更新）

### 待做（本周）
- [ ] [选题 A] - 信号来源：XXX - 预计产出：视频 - 截止：06-28
- [ ] [选题 B] - 信号来源：XXX - 预计产出：文章 - 截止：06-30

### 待定（队列）
- [ ] [选题 C] - 信号来源：XXX - 打分：3.2 - 备注：等热点发酵
- [ ] [选题 D] - 信号来源：XXX - 打分：2.8 - 备注：产能有余力再做

### 已完成
- [x] [选题 E] - 产出链接：XXX - 数据：播放 XX / 点赞 XX

### 已放弃
- [~] [选题 F] - 放弃原因：热点已过 / 竞争太激烈
```

### 选题池维护原则

1. **先做 A 级信号**——时间窗口不等人
2. **每周清一次**——过期选题果断删
3. **记录放弃原因**——避免重复踩坑
4. **爆款要复盘**——记录什么选题跑得好

## 七、选题库 vs 选题池

| 概念 | 定义 | 用途 |
|------|------|------|
| **选题池** | 待做的具体选题 | 日常生产 |
| **选题库** | 可持续做的选题方向 | 长期规划 |

### 选题库示例（长期方向）

| 方向 | 用户是谁 | 解决什么问题 | 内容形式 |
|------|---------|-------------|---------|
| AI 工具测评 | 想用 AI 的人 | 选工具、避坑 | 视频 + 文章 |
| 自动化教程 | 想提效的人 | 搭建工作流 | 视频 |
| 赚钱案例拆解 | 想副业的人 | 找方向、学方法 | 文章 |
| ... | ... | ... | ... |

**关系**：选题库是「菜谱」，选题池是「今天做什么菜」。从选题库取方向，结合当下信号，产出具体选题。

## 八、避坑指南

### 不要做的选题

| 类型 | 为什么不做 |
|------|----------|
| 纯情绪宣泄 | 没有信息增量，用户看完就忘 |
| 过于小众 | 流量天花板太低 |
| 和自己定位不符 | 吸引来的粉丝不精准 |
| 只有一次需求 | 做完没有复利 |
| 明显违反平台规则 | 发不出来 |

### 常见错误

1. **追热点不转化**——照着别人做，没有自己的角度
2. **选题池太满**——列了 100 个，一个没做
3. **只看数据不看匹配**——数据好的选题不一定适合自己
4. **忽视评论区**——最好的选题在用户的问题里
