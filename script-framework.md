# 文案生产框架

> 全链路内容生产机器的「文案」环节实现方案。输入选题，输出适配多平台的视频文案。

## 定位

这是 framework.md 中「文案 → ⏳」环节的具体实现。目标不是写出诺贝尔文学奖，而是在 15 分钟内产出一条「能涨粉」的文案。

## 核心理念

1. **文案为视频服务**——不是写文章，是写「配音脚本」。每句话都要能在 3 秒内被耳朵理解。
2. **平台决定结构**——抖音要前 3 秒 hook，B 站要信息增量，视频号要情绪共鸣，YouTube 要 SEO 前置。
3. **AI 写初稿，人改关键**——AI 负责结构和信息组织，人只动关键 20%（钩子、节奏、情绪锚点）。

## 文案生产流水线

```
选题 → 钩子 → 骨架 → 填充 → 润色 → 定稿
```

### 第 1 步：钩子工程（5 分钟）

**目的**：用户刷到的前 3 秒决定划走还是留下。

| 钩子类型 | 公式 | 示例 |
|---------|------|------|
| **反常识** | 「你以为 X 是这样？错了」 | 「你以为 AI 视频很贵？错了」 |
| **痛点直击** | 「你是不是也遇到 X 问题？」 | 「你是不是也剪了 3 小时没人看？」 |
| **结果前置** | 「我用了 X 方法，30 天涨了 Y 粉」 | 「我用这套框架，7 天出了 3 条爆款」 |
| **悬念** | 「关于 X，99% 的人都不知道这 1 点」 | 「关于小红书流量，99%的人都不知道这个算法漏洞」 |
| **身份标签** | 「如果你是 X 人群，这条必看」 | 「如果你在做自媒体副业，这条必看」|

**钩子选择原则**：
- 抖音/B 站/视频号：前 3 秒必须有一个钩子，画面 + 语音双重 hook
- YouTube：标题 + 前 30 秒做钩子，前 3 秒依然重要但更依赖搜索流量
- 钩子与内容强相关，不要做标题党——用户被 hook 进来发现被骗，完播率崩盘

### 第 2 步：骨架搭建（3 分钟）

**用 AI 输出结构**。Prompt 公式：

```
选题：[XXX]
目标平台：[抖音/B站/视频号/YouTube]
目标时长：[30s/60s/3min/10min]
输出要求：
1. 给我 3 个不同的开头钩子
2. 给出正文骨架（3-5 点，每点一句话核心论点）
3. 每个论点配一个论据/案例
4. 给出结尾方案（CTA/总结/悬念续集）
```

**骨架原则**：
- 短视频（< 60s）：1 个钩子 + 1 个核心信息 + 1 个 CTA = 信息密度最大化
- 中视频（3-10min）：1 个钩子 + 3-5 个论点（递进关系）+ 结尾总结
- 长视频（> 10min）：1 个钩子 + 叙事线（问题 → 探索 → 方案）
- 每 15-30 秒换一个画面或信息点，防止注意力滑坡

### 第 3 步：填充文案（5 分钟）

**核心规则：写口语，不是写文章**

| 错误（书面语） | 正确（口语） |
|-------------|-----------|
| 近年来，随着人工智能技术的不断发展 | 这两年 AI 发展特别快 |
| 我经过反复实践得出以下结论 | 我试了 20 次，发现这么干最好用 |
| 该产品具备三大核心优势 | 这东西有 3 个好处 |

**口语化技巧**：
1. **写完后念一遍**——念不顺的地方就是需要改的地方
2. **短句优先**——一句不超过 20 个字，超出就拆
3. **用「你」而不是「用户」**——「你会发现」代替「用户会发现」
4. **模拟停顿**——用「……」「对吧」「你想想」制造呼吸节奏
5. **情绪词适度**——「太离谱了」「真的绝了」「我直接傻掉」，但要自然

**AI Prompt 技巧**：
```
请把以下文案改成口语化视频脚本。要求：
- 每句不超过 20 字
- 用「你」直接与观众对话
- 加入自然的停顿和语气词
- 适合[抖音/B站/视频号]风格
- 标注每段对应的画面建议

文案：[粘贴]
```

### 第 4 步：润色 + 卡点（2 分钟）

检查清单（自查用）：

- [ ] 有没有一个清晰的钩子在前 3 秒？
- [ ] 语速测试：正常语速念一遍，是否在目标时长 ±10% 内？
- [ ] 有没有「废话段落」可以整段删除？（删掉后意思不变就是废话）
- [ ] 每 15-30 秒是否有一个「信息刺激点」？（新的观点/反转/数据/案例）
- [ ] 结尾有没有明确的 CTA？（关注/点赞/评论/下期预告）
- [ ] 有没有「听不懂的术语」需要改成大白话？
- [ ] 有没有「自嗨内容」？（只有你感兴趣、观众无感的部分）
- [ ] 文案是否适配目标平台的调性？

## 多平台适配对照

| 维度 | 抖音 | B站 | 视频号 | YouTube |
|------|------|-----|--------|---------|
| 时长偏好 | 15-60s | 3-15min | 30s-3min | 8-20min |
| 钩子速度 | 0.5s 内必须 hook | 3-5s 内 hook | 1-2s 内 hook | 标题 SEO 占 50% |
| 语速 | 快（每分钟 280-320 字）| 适中（220-260）| 适中偏慢 | 正常（160-200） |
| 情绪 | 夸张/反转/冲突 | 理性+幽默 | 共鸣/温情 | 专业/信息量 |
| 标题风格 | 情绪化+悬念 | 信息增量 | 共鸣+价值 | SEO+搜索关键词 |
| CTA 强度 | 强（点赞关注收藏）| 适中 | 自然引导 | 订阅+看下一期 |

## AI 文案工具组合推荐

| 工具 | 强项 | 弱项 |
|------|------|------|
| Claude Sonnet 4 | 结构清晰、逻辑强、适合 B 站/YouTube | 节奏偏正经，需要手动加口语感 |
| DeepSeek V3 | 中文口语化好、成本低 | 长文结构有时松散 |
| 豆包 | 中文语境理解好、小红书风格适配 | 创意性一般 |
| ChatGPT o3-mini | 创意强、钩子生成好 | 中文口语化弱于 DeepSeek |

**推荐工作流**：DeepSeek 写初稿 → Claude 优化结构 → 人做钩子和关键调整 → 豆包改短文案适配

| 阶段 | 当前目标 | 终极目标 |
|------|---------|---------|
| 选题 → 骨架 | 5 min | 2 min |
| 骨架 → 初稿 | 5 min | 3 min |
| 初稿 → 定稿 | 5 min | 3 min |
| **总耗时** | **15 min/条** | **8 min/条** |
