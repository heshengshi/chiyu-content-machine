# AI 工具测评方法论

> 系统化测评 AI 工具的框架。不是「用一下觉得好」，而是「按维度打分 → 产出可复用的测评结论」。

## 为什么需要这个框架

**问题**：看到新工具就试，试完就忘。测评结论零散，无法复用，无法对比。

**目标**：
- 每测一个工具，产出一份结构化测评报告
- 测评结论可直接转化为内容（测评视频/文章）
- 多个工具可横向对比，辅助决策

---

## 一、测评维度矩阵

一个 AI 工具值不值得用，从这 6 个维度打分：

| 维度 | 权重 | 评估什么 | 打分标准（1-5分）|
|------|------|---------|-----------------|
| **核心功能** | 30% | 它承诺的功能能不能做到？ | 1=完全不行 3=勉强能用 5=超出预期 |
| **上手成本** | 20% | 新用户多久能产出第一个结果？ | 1=要学1天 3=30分钟 5=打开就会 |
| **输出质量** | 20% | 产出的东西能不能直接用？ | 1=要大改 3=小改能用 5=直接发布 |
| **成本效率** | 15% | 免费/付费？性价比如何？ | 1=很贵且不值 3=价格合理 5=免费或超值 |
| **稳定性** | 10% | 会崩吗？结果一致吗？ | 1=经常崩 3=偶尔抽风 5=很稳 |
| **生态位** | 5% | 有替代品吗？不可替代性？ | 1=满街替代品 3=有差异点 5=独一无二 |

**打分公式**：加权总分 = Σ(维度分 × 权重)，满分 5 分

**决策规则**：
- ≥ 4.0 分 → 推荐使用，可纳入生产流程
- 3.0-3.9 分 → 备选，特定场景用
- < 3.0 分 → 不推荐，除非没有替代品

## 二、测评流程（单工具）

```
发现工具 → 快速验证 → 深度测试 → 对比定位 → 输出结论
   5min       15min      30min      10min      10min
```

### Step 1：快速验证（5 分钟）

**目的**：快速判断这个工具值不值得深度测。

**操作**：
1. 打开工具官网，看它承诺什么功能
2. 找到「快速开始」或「Demo」，跑一遍
3. 看一眼定价（免费/免费额度/付费）
4. 搜索一下有没有明显差评（Google/GitHub Issues）

**判断**：
- 核心功能 5 分钟内跑不通 → 直接放弃，记录「放弃原因」
- 有大量「已死」「不维护」反馈 → 直接放弃
- 跑通了 → 进入深度测试

### Step 2：深度测试（30 分钟）

**核心原则**：用真实任务测，不用官方 Demo 测。

**测试任务库**（按工具类型选 3 个任务）：

| 工具类型 | 测试任务 A | 测试任务 B | 测试任务 C |
|---------|-----------|-----------|-----------|
| 文案生成 | 写一条抖音带货文案（美妆） | 写一篇 B 站视频脚本（AI 工具测评） | 把技术文章改成口语化 |
| 图像生成 | 生成一张小红书封面（AI 工具主题） | 生成一张公众号配图（抽象概念） | 生成一致风格的多张图 |
| 视频生成 | 用文案生成 30s 短视频 | 给定图片生成动态效果 | 批量生成视频封面 |
| 音频/TTS | 朗读一段技术文案 | 模仿特定音色 | 多角色对话生成 |
| 代码工具 | 写一个爬虫脚本 | 重构一段烂代码 | 解释一段复杂代码 |

**测试记录表**：

```
工具名称：[XXX]
测试时间：[YYYY-MM-DD]
测试任务：
  - 任务 A：[结果] | 耗时 [X]min | 评分 [X]/5
  - 任务 B：[结果] | 耗时 [X]min | 评分 [X]/5
  - 任务 C：[结果] | 耗时 [X]min | 评分 [X]/5
发现的问题：
  - [问题 1]
  - [问题 2]
惊喜点：
  - [惊喜 1]
```

### Step 3：对比定位（10 分钟）

**目的**：这个工具在同类里的位置。

**操作**：
1. 列出 2-3 个竞品（已知或搜索找到）
2. 快速对比核心差异（不是重新测，是凭已有认知对比）
3. 定位：这个工具适合什么人？什么场景？

**对比表模板**：

| 维度 | 本工具 | 竞品 A | 竞品 B |
|------|--------|--------|--------|
| 核心优势 | | | |
| 明显短板 | | | |
| 适合人群 | | | |
| 定价 | | | |

### Step 4：输出结论（10 分钟）

**产出物**：一份结构化测评结论

```markdown
## [工具名称] 测评结论

**总分**：[X.X]/5.0
**定位**：[一句话定位，如「最适合新手的 AI 视频工具」]
**推荐度**：⭐⭐⭐⭐☆（推荐/备选/不推荐）

### 适合谁
- [人群 A]
- [人群 B]

### 不适合谁
- [人群 C]

### 核心优势
1. [优势 1]
2. [优势 2]

### 明显短板
1. [短板 1]
2. [短板 2]

### 使用建议
[具体场景下的使用建议，如「做抖音短视频可以，做 B 站长视频不行」]

### 替代品
```

## 三、横向对比测评（多工具）

当需要从多个工具中选一个时，做横向对比。

**场景**：比如「选一个 AI 视频生成工具」，候选有 Runway、Pika、Kling、Sora

**方法**：
1. 用**同一个测试任务**测所有候选工具
2. 按维度矩阵打分
3. 输出对比表

**对比任务选择原则**：
- 必须是真实任务，不是各工具的 Demo
- 任务要能覆盖所有候选工具的核心功能
- 任务要有代表性（你实际会用的场景）

**输出格式**：

| 工具 | 总分 | 核心功能 | 上手成本 | 输出质量 | 成本效率 | 稳定性 | 结论 |
|------|------|---------|---------|---------|---------|--------|------|
| Runway | 4.2 | 5 | 3 | 4 | 3 | 4 | 专业首选 |
| Pika | 3.8 | 4 | 4 | 4 | 4 | 3 | 新手友好 |
| Kling | 4.0 | 4 | 4 | 5 | 5 | 3 | 性价比之王 |
| Sora | 4.5 | 5 | 3 | 5 | 2 | 5 | 质量天花板但贵 |

**推荐结论**：新手用 Kling，专业用 Runway，不差钱用 Sora。

## 四、测评结论的内容化

**测评本身就可以变成内容**。一条测评视频/文章的价值：

1. **节省他人时间**——别人不用自己测了
2. **建立专业形象**——持续测评 = 这个领域的测评者
3. **变现路径**——测评 → 联盟链接/广告/咨询

**测评内容模板**：

### 视频脚本版（60s）

```
[钩子 3s]
「我测了 10 个 AI 视频工具，只有 3 个值得用」

[正文 45s]
「第一个是 [工具 A]，[核心优势]，但[短板]，适合[人群]
第二个是 [工具 B]，[核心优势]，但[短板]
第三个是 [工具 C]，[核心优势]

其他 7 个为什么不推荐？[一句话概括共同问题]」

[CTA 12s]
「完整测评报告放评论区了，关注我，每周测一个新工具」
```

### 文章版（小红书/公众号）

```
标题：「[工具名] 深度测评：值不值得买？」

结构：
1. 一句话结论（推荐/不推荐）
2. 这个工具是干嘛的
3. 我怎么测的（测试任务）
4. 测评结果（维度打分表）
5. 适合谁/不适合谁
6. 和竞品比怎么样
7. 最终建议

配图：
- 工具界面截图
- 输出结果对比图
- 打分表图
```

## 六、测评原则

1. **真实任务测试**——不用官方 Demo，用自己实际会遇到的场景
2. **横向对比才有意义**——单独测一个工具很难判断好坏，要和竞品比
3. **记录放弃原因**——放弃的工具也要记录为什么放弃，避免重复试错
4. **测评即内容**——每次测评都产出可发布的内容，一鱼两吃
5. **定期复测**——AI 工具迭代快，3 个月前的测评可能已过时
