# 赚钱机会双视角挖掘框架

> 从两个视角扫描市场，找出需求信号和赚钱机会。

## 一、生产者视角：看同行在做什么

**核心问题**：已经赚到钱的人在卖什么？怎么卖的？

### 观察维度

| 维度 | 观察什么 | 信号类型 |
|------|---------|---------|
| **广告** | 投了什么广告？文案怎么写的？落地页长什么样？ | 高利润产品 |
| **内容** | 发什么内容？哪些内容互动高？更新频率？ | 流量密码 |
| **产品** | 核心卖点是什么？定价策略？竞品在哪？ | 产品方向 |
| **链路** | 从引流到成交中间几步？用了什么工具？ | 可复制模式 |
| **招聘** | 在招什么岗位？说明在扩张什么方向 | 行业热点 |

### 平台

Boss直聘(招聘信息) / 抖音/小红书/视频号(广告+内容) / 微信公众号 / GitHub Trending / ProductHunt / X(Twitter)

## 二、消费者视角：看用户在烦什么

**核心问题**：用户在抱怨什么？愿意为什么付钱？

### 观察维度

| 维度 | 观察什么 | 信号类型 |
|------|---------|---------|
| **评论区** | 用户的真实反馈和补充需求 | 未被满足的需求 |
| **投诉/差评** | 对现有产品的不满 | 竞品改进机会 |
| **求助帖** | 直接问"怎么解决X" | 刚需 |
| **比较帖** | "A和B哪个好" | 决策帮助需求 |
| **理想描述** | "要是能X就好了" | 产品灵感 |

### 平台

Reddit / 知乎 / 小红书评论区 / 觅游求助虾 / AI Native群 / 产品评价区

## 三、需求信号评分卡

每条需求信号用这个评分卡打分：

| 指标 | 权重 | 1分 | 3分 | 5分 |
|------|------|-----|-----|-----|
| **信号频率** | 30% | 1人提过 | 5-10人提过 | 50+人提过 |
| **付费意愿** | 30% | 想要免费的 | 愿意付小额 | 已有人付高价 |
| **解决难度** | 20% | 很复杂 | 一般 | 很快能做出来 |
| **与我匹配** | 20% | 不相关 | 有点相关 | 直接能用 |

## 五、使用方法

当你发现一个赚钱机会/需求信号时，用这个流程分析：

```
1. 这是消费者视角还是生产者视角发现的信号？
2. 用评分卡打分（0-30分，25+可做）
3. 对齐到三个项目：个人IP/娱乐/工具
4. 输出一个"最小验证方案"（1页，不能再多）
```
