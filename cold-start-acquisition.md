# 冷启动获客手册：从 0 到第一个付费客户的战术 playbook

> **定位**：`offer-design-framework.md` 教你设计 Offer，`first-month-survival-guide.md` 教你规划第一月节奏——但**怎么从"我有 Offer 但没人认识我"到"有人付钱了"**？这份手册填的就是这个缺口。
>
> **核心主张**：冷启动不是等流量来，是主动把 Offer 塞到有需求的人面前。获客不是营销，是**匹配**——你不需要说服不需要的人，你需要找到已经需要的人。
>
> **适用场景**：没有作品集、没有粉丝、没有口碑、没有预算——只有你自己 + AI + 一个设计好的 Offer。
>
> **与现有框架的关系**：
> - 上游：`offer-design-framework.md`（设计好 Offer）→ 本手册（把 Offer 卖出去）
> - 下游：`project-execution-framework.md`（交付后标准化放大）
> - 并行：`community-patterns.md`（1-3-10 阶梯）→ 本手册是 1 的战术拆解

## 一、冷启动的本质：不是营销，是匹配

### 1.1 为什么"发内容等客户来"在冷启动阶段无效

```
有粉丝的人：发内容 → 粉丝看到 → 信任已有 → 咨询 → 成交
没粉丝的人：发内容 → 没人看到 → 没信任 → 没咨询 → 没成交
```

**冷启动阶段，内容是弹药，主动触达是枪。** 只有弹药没有枪，打不中任何目标。

### 1.2 冷启动的三个核心动作

```
1. 找到已经有需求的人（不是创造需求，是发现需求）
2. 用最小成本证明你能解决（不是说服，是展示）
3. 降低对方决策门槛（不是降价，是降风险）
```

### 1.3 冷启动获客漏斗

```
潜在需求池（1000+ 人有这个需求）
    ↓ 主动触达
触达池（你主动接触了 50 人）
    ↓ 回应率 20%
对话池（10 人回复了你）
    ↓ 展示能力 + 降风险
意向池（3 人表示感兴趣）
    ↓ 促单
成交池（1 人付钱）
```

**关键指标**：
- 触达→回应率：目标 > 15%（低于 = 触达方式/话术有问题）
- 对话→意向率：目标 > 30%（低于 = 能力展示/信任不够）
- 意向→成交率：目标 > 30%（低于 = 定价/风险逆转有问题）

## 二、找到已经有需求的人：5 个高转化渠道

### 2.1 渠道一：社群求助帖（转化率最高）

**原理**：在群里主动问问题的人，需求最明确、最急迫。

**操作**：

| 步骤 | 动作 | 话术模板 |
|------|------|---------|
| 1 | 每天扫 AI Native 群/生财群/树成林群的求助帖 | — |
| 2 | 筛选你能 3 天内交付的需求 | — |
| 3 | 私聊发话术 | "看到你在问 XX，我刚好做过这个，要不要我帮你做一个样例看看？不收钱，你觉得有用再聊" |

**关键**关键**：第一条消息不卖，只帮。帮了之后对方自然问"多少钱"——这时候才是卖。**

**每日动作**：扫 3 个群，找 5 个求助帖，私聊 3 个。一周 15 个触达，预期 2-3 个对话。

### 2.2 渠道二：闲鱼/淘宝服务对标（最快验证定价）

**原理**：闲鱼上已经有人在卖类似服务 = 需求已被验证 + 定价有参考。

**操作**：

1. 搜索关键词："AI 代做""AI 帮做""AI 文案""AI 视频""AI 小红书"
2. 找到销量 > 10 的同类服务
3. 分析他们的：定价、交付物描述、图片、评价
4. **做一个比他们更好的主图 + 更清晰的交付物描述**
5. 挂上去，定价 = 对标价的 70%（冷启动折扣）

**定价参考**（2026 年 7 月闲鱼 AI 服务市场）：

| 服务类型 | 市场价 | 冷启动价 | 交付周期 |
|---------|--------|---------|---------|
| AI 文案代写 | 50-200/篇 | 30-100/篇 | 1 天 |
| AI 配图生成 | 30-100/组 | 20-60/组 | 1 天 |
| AI 工作流搭建 | 200-500/个 | 150-300/个 | 3 天 |
| AI 视频制作 | 100-300/条 | 80-200/条 | 2-3 天 |
| AI 内容方案 | 300-800/份 | 200-500/份 | 2 天 |

**每日动作**：维护 3 个闲鱼商品，每天更新 1 个商品的主图或描述。

### 2.3 渠道三：小红书/抖音评论区（流量最大但转化低）

**原理**：爆款内容下有大量"怎么做""求教程"的评论 = 需求池。

**操作**：

1. 搜索"AI + 你的服务关键词"
2. 找到近 7 天的爆款（点赞 > 1000）
3. 在评论区找"怎么做""求教程""能帮我做吗"的评论
4. 私聊回复："我看到你在问 XX，我刚好做这个，发你一个样例看看？"

**注意**：
- 不要在评论区直接卖（容易被举报/封号）
- 私聊前先看对方主页，判断是否是目标用户
- 第一条私聊只提供价值，不报价

**每日动作**：找 3 个爆款帖，私聊 5 个评论者。

### 2.4 渠道四：朋友圈/私域（信任最高但量小）

**原理**：微信好友已经认识你，信任成本最低。

**操作**：

1. **不要发广告**——发"我在做什么 + 刚帮谁做了什么 + 效果如何"
2. 格式：`[结果] + [过程一句话] + [如果你也需要，私聊我]`
3. 示例："刚用 AI 帮一个做小红书的朋友做了 10 条图文，3 天涨了 200 粉。如果你也需要，私聊我"

**频率**：每周 2-3 条，穿插在正常朋友圈中，不要连续发。

**关键**：朋友圈获客的核心不是广撒网，是让对的人知道你在做这件事。2100 好友中，只要有 5 个人有需求，就够第一单了。

### 2.5 渠道五：生财有术/知识星球帖子（精准但需要刷脸）

**原理**：生财有术的帖子评论区 = 高质量需求池。

**操作**：

1. 找到"项目复盘""赚钱结果"类帖子
2. 评论区问有价值的问题（不是"求带"，是"你做 XX 环节时怎么解决 YY 问题的？"）
3. 帖子作者回复你后，私聊深入聊
4. 聊的过程中自然展示你的能力
5. 如果对方有你需要解决的问题，主动提出帮忙

**每日动作**：在生财有术评论 2 个帖子，私聊 1 个有互动的人。

## 三、主动触达话术系统：从第一条消息到成交

### 3.1 第一条消息的 3 个原则

1. **不卖**——第一条消息只提供价值或建立连接
2. **具体**——说清楚你能帮什么，不是"我能帮你做很多"
3. **低压力**——对方不需要立刻做决定

### 3.2 五种开场话术模板

#### 模板 A：帮人型（适用于社群求助帖）

```
看到你在问 [具体问题]，我刚好做过这个。
[一句话说你的经验/结果]
要不要我帮你 [最小帮助动作]？不收钱，你觉得有用再聊。
```

示例：
> 看到你在问怎么用 AI 批量做小红书图文，我刚好用 AI 帮自己做了 30 条，跑出了 200 粉。要不要我帮你做 1 条样例看看？不收钱，你觉得有用再聊。

#### 模板 B：请教型（适用于你想链接的人）

```
[对方名字/称呼]，看了你 [具体内容]，有个问题想请教：
[一个具体的、有深度的问题]
我自己在做 [你的方向]，遇到了 [具体卡点]。
```

示例：
> 哥，看了你发的 AI 视频自动化复盘，有个问题想请教：你用 social-auto-upload 发布时，小红书的审核延迟怎么处理的？我自己在做 AI 内容矩阵，卡在发布后经常被限流。

#### 模板 C：展示型（适用于朋友圈/公开渠道）

```
[具体结果] + [一句话过程] + [如果你也需要]
```

示例：
> 刚用 AI 帮朋友做了 10 条小红书图文，3 天涨了 200 粉。核心是用 AI 生成文案 + Midjourney 出图 + 排期表批量发。如果你也需要，私聊我。

#### 模板 D：对标型（适用于闲鱼/淘宝）

```
[服务名] | [核心结果承诺]
✅ 包含：[3-5 个具体交付物]
✅ 不包含：[明确边界]
✅ 保障：[风险逆转]
💰 [价格]（前 X 名 [折扣价]）
```

示例：
> AI 小红书图文包 | 7 天产出 10 条精准内容
> ✅ 包含：10 条文案 + 10 组配图 + 1 份发布排期表
> ✅ 不包含：账号运营/投放/客服
> ✅ 保障：不满意全额退款
> 💰 ¥599（前 3 名 ¥299）

#### 模板 E：合作型（适用于想长期合作的人）

```
我在做 [你的方向]，看到你也在做 [对方方向]。
[一个具体的互补点]。
要不要试试 [最小合作动作]？
```

示例：
> 我在做 AI 内容自动化，看到你也在做小红书运营。我出内容生产能力，你出运营经验，要不要试试一起做 1 个账号？我负责内容，你负责运营，分成聊。

### 3.3 对话推进：从第一条消息到成交的 4 步

```
Step 1：建立连接（第一条消息）
  → 目标：对方回复你
  → 检验：对方回了，哪怕只回"好的"

Step 2：展示能力（免费做一个小东西）
  → 目标：对方认可你的能力
  → 检验：对方说"不错"/"可以"/"挺好的"

Step 3：提出 Offer（对方认可后自然过渡）
  → 目标：对方了解你的服务内容和价格
  → 话术："如果你需要更多，我可以 [Offer 描述]，价格是 [价格]"
  → 检验：对方问细节或表示考虑

Step 4：促单（对方犹豫时）
  → 目标：成交
  → 话术："这样吧，[风险逆转承诺]，你觉得行的话我们开始？"
  → 检验：对方付款或确认开始
```

### 3.4 常见拒绝及应对

| 拒绝 | 真实含义 | 应对 |
|------|---------|------|
| "太贵了" | 不确定值不值 | "理解，这样吧，我先帮你做 1 个小样例，你觉得值再聊价格" |
| "我再想想" | 还不信任你 | "没问题，我发你几个我做的案例看看，你参考一下" |
| "不需要" | 真不需要 or 没听懂 | "好的，如果你以后有需要随时找我"（不纠缠，保持关系） |
| "别人更便宜" | 在比价 | "可以理解，不过我的服务包含 [差异化点]，你可以先看看效果再决定" |
| "我怕做不好" | 怕风险 | "我承诺不满意全额退款，你没有任何风险" |

## 四、信任构建：没有案例时怎么办

### 4.1 四种零案例信任策略

#### 策略 1：做自己的案例

```
用自己的账号/业务做实验 → 拿自己的数据说话

具体动作：
1. 用 AI 帮自己做 10 条小红书内容
2. 记录数据：发布时间、阅读量、点赞、评论
3. 3 天后截图数据
4. 把过程和结果做成案例展示
```

**优势**：最真实、最快速、零成本
**劣势**：数据可能不够亮眼

#### 策略 2：免费帮 1 个人做

```
在群里找 1 个有需求的人 → 免费帮他做 → 换取真实评价 + Before-After

具体动作：
1. 在 AI Native 群/生财群找 1 个正在做小红书/内容的人
2. 私聊："我正在打磨我的 AI 内容服务，需要 1 个真实用户测试。我免费帮你做 5 条内容，你给我真实反馈就行"
3. 做完交付，要 3 样东西：
   - 真实评价（好不好、哪里可以更好）
   -Before-After 数据截图
   -如果满意，允许你用这个案例做展示
```

**优势**：有第三方背书
**劣势**：需要 2-3 天时间

#### 策略 3：公开过程

```
把做的过程发出来 → 让潜在客户看到你的专业度

具体动作：
1. 每天发 1 条"我在做什么"的内容
2. 展示：工具选择、prompt 设计、生成过程、调优过程、最终效果
3. 渠道：朋友圈 + 小红书 + 生财有术帖子
4. 格式："Day X：用 AI 做 [具体事]，今天解决了 [具体问题]，效果是 [具体数据]"
```

**优势**：持续积累信任、内容本身就是获客工具
**劣势**：见效慢，需要持续

#### 策略 4：承诺退款

```
第一单承诺"不满意全额退款" → 把风险从客户转移到自己

具体动作：
1. 在 Offer 中明确写"不满意 7 天内全额退款"
2. 交付前确认需求，避免理解偏差
3. 交付后主动问反馈
4. 如果不满意，立刻退款，不纠缠
```

**优势**：最直接降低决策门槛
**劣势**：可能被白嫖（但概率很低，因为沟通成本也是成本）

### 4.2 信任构建时间线

```
Day 1-2：做自己的案例（策略 1）
Day 2-3：免费帮 1 个人做（策略 2）
Day 1-7：每天发 1 条过程内容（策略 3）
Day 3+：所有 Offer 附带退款承诺（策略 4）
```

## 六、冷启动的 7 个致命错误

### 错误 1：等流量来

```
❌ "我先做 30 条内容，等粉丝多了再卖"
✅ "我做 1 条内容的同时，主动找 5 个可能需要的人"

原因：冷启动阶段，内容是弹药不是枪。没有主动触达，再好的内容也没人看到。
```

### 错误 2：第一条消息就卖

```
❌ "你好，我是做 AI 内容服务的，价格 599，需要吗？"
✅ "你好，看到你在问 XX，我刚好做过，要不要我帮你做一个样例看看？"

原因：没有信任基础时，直接卖 = 推销 = 被拒绝。先帮后卖，转化率高 10 倍。
```

### 错误 3：只在一个渠道

```
❌ "我只在闲鱼挂了"
✅ "闲鱼 + 社群私聊 + 朋友圈 + 评论区，4 个渠道同时铺"

原因：单一渠道的流量有限。4 个渠道各触达 10 人 = 40 人触达，比 1 个渠道触达 40 人容易。
```

### 错误 4：触达了但不跟进

```
❌ 私聊了 10 个人，3 人，对方回了但没继续聊
✅ 对方回复后，24 小时内推进 1 步

原因：对方回复你 = 窗口期。窗口期很短，不跟进 = 白触达。
```

### 错误 5：定价太高或太低

```
❌ "第一单 9.9"（太低 = 不值钱感知）
❌ "第一单 2000"（太高 = 没信任支撑）
✅ "第一单 = 市场价 30%-50%，但原价标清楚"（折扣有理由，原价锚定价值）

原因：9.9 吸引的是白嫖用户，反馈质量差。2000 没有信任支撑，没人买。中间档 + 折扣理由 = 最优。
```

### 错误 6：没有案例就硬卖

```
❌ "我很专业，相信我"
✅ "我还没案例，但我可以先免费帮你做 1 个小样例，你觉得好再聊"

原因：冷启动阶段，"我很专业"没有任何说服力。用行动证明，不用嘴证明。
```

### 错误 7：被拒绝后不再触达

```
❌ 被拒 3 次后不再主动找人了
✅ 被拒是正常的，冷启动转化率 1%-5%。触达 100 人成交 1-5 单 = 正常

原因：冷启动的转化率本来就很低。不是你不行，是概率问题。提高触达量 = 提高成交概率。
```

## 七、从第一个客户到 1-3-10 阶梯

### 7.1 第 1 个客户：验证闭环

```
目标：跑通"发现需求→设计交付→找到人→对方付钱→你交付→对方满意→你拿到钱"
定价：市场价 30%-50%
关键：超预期交付 + 要评价 + 要转介绍
```

### 7.2 第 2-3 个客户：排除偶然

```
目标：证明第 1 单不是运气
方法：
1. 让第 1 个客户推荐（"你身边有没有人也需要的？"）
2. 在新渠道曝光（第 1 单在社群，第 2 单在闲鱼）
3. 用第 1 单的案例做信任背书
定价：市场价 50%-70%
关键：3 个客户中至少 1 个是陌生人（非亲友）
```

### 7.3 第 4-10 个客户：验证可复制

```
目标：证明获客和交付可以复制
方法：
1. 建立稳定获客渠道（哪个渠道效果最好，加大量）
2. 标准化交付流程（SOP + 模板）
3. 提价到市场价 70%-100%
关键：10 个客户中至少 5 个是自己找来的（不是你主动找的）
→ 这说明你的内容/口碑已经在帮你获客了
```

### 7.4 10+ 客户：进入 project-execution-framework

```
当获客和交付都稳定后，进入 project-execution-framework.md 的系统化阶段：
- 交付 SOP 化
- 获客内容化
- 考虑从服务→产品进化
```

## 八、冷启动获客的 Quick-Cheap-Easy Test 清单

每次获客动作前，用 `process-result-feedback-layer` 的测试模板：

```text
1. 我的假设：[渠道]的[话术]能获得[预期回应率]的回复
2. 为什么可能成立：[理由]
3. 最便宜的测试：发 10 条消息，看回复率
4. 预期反馈：≥ 2 人回复
5. 成败判据：回复率 ≥ 15% = 继续；< 15% = 换话术或换渠道
6. 反馈不符合预期时更新：话术/渠道/目标人群的判断
```

**具体测试优先级**：

| # | 测试 | 成本 | 预期时间 | 成功标准 |
|---|------|------|---------|---------|
| 1 | 社群私聊 10 人（帮人型话术） | 0 元 | 1 天 | ≥ 2 人回复 |
| 2 | 闲鱼挂 1 个商品 | 0 元 | 2 天 | ≥ 1 个咨询 |
| 3 | 朋友圈发 1 条过程展示 | 0 元 | 1 天 | ≥ 1 个私聊 |
| 4 | 小红书评论区私聊 5 人 | 0 元 | 1 天 | ≥ 1 人回复 |
| 5 | 免费帮 1 人做小样例 | 2-3 小时 | 2 天 | 对方满意 + 允许展示 |

**测试顺序**：1 → 3 → 4 → 2 → 5。先做成本最低、最快的。
